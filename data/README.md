# City dataset

`cities.json` is the single source of truth for all city pages, country pages, time-difference pages, and search. The server loads it once at startup and caches it in memory for fast lookups. `countries.json` groups city slugs by country (e.g. for reference or tooling); the app builds its country index from `cities.json`.

## AI-generated city content (overview, history, FAQs, great places)

City pages show **overview**, **history**, **FAQs**, and **Great places to visit** when a file exists at `data/city-content/<slug>.json`. Without it, only generic place chips and basic time info are shown.

- **Full content** (overview, history, economy, tourism, best_places, best_restaurants, FAQs):  
  `OPENAI_API_KEY=sk-... node scripts/generateCityContent.js [--major-first] [--limit N]`  
  Use `--major-first` so major world cities (and popular cities) are processed first, then the rest. Use `--limit N` to cap how many cities to process.
- **Great places only** (lighter, one API call per city):  
  `OPENAI_API_KEY=sk-... node scripts/generatePlaces.js [--major-first] [--limit N]`  
  Use `--major-first` to fix “Great places” for nearby and major cities first.
- **Fix all cities at once** (no limit; run in background; takes a long time):  
  `OPENAI_API_KEY=sk-... npm run generate-city-content:major-first` for full content, or  
  `OPENAI_API_KEY=sk-... npm run generate-places:major-first` for places only.

NPM scripts: `generate-city-content`, `generate-city-content:major-first`, `generate-places`, `generate-places:major-first`.

## Regenerating the dataset (5000+ cities)

The dataset is generated by a **pipeline**, not by hand.

1. **Run the generator** (downloads GeoNames data, filters, builds slugs, writes JSON):
   ```bash
   node scripts/generateCities.js
   ```
   Or: `npm run generate-cities` (if added to package.json).

2. **Requirements:** Node.js 14+, and `unzip` (macOS/Linux) or `7z` on PATH to extract the cities archive.

3. **Options (env):**
   - `MIN_POPULATION=100000` — only cities with population ≥ this (default 100000). Use a lower value (e.g. `50000`) to get more cities.
   - `GEONAMES_CITIES_URL` — override cities source (default: GeoNames `cities5000.zip`).
   - `GEONAMES_COUNTRIES_URL` — override country info (default: `countryInfo.txt`).

4. **Output:** `data/cities.json` (5000+ entries) and `data/countries.json`. Restart the server to pick up changes.

## How to add new cities

- **Bulk:** Re-run the pipeline (see above). To include more cities, lower `MIN_POPULATION` or point to a larger source (e.g. `cities1000.zip` or `allCountries.zip` from GeoNames), then re-run.
- **By hand:** Append a record to `cities.json` with at least `city`, `country`, `slug`, `timezone`. Use the same slug rules as below so URLs stay consistent. Restart the server after editing.

## How to update slugs safely

Slugs are used in URLs (`/time/:citySlug`, `/time-difference/:a/:b`). Changing a slug breaks existing links and bookmarks.

- **Best:** Regenerate with the pipeline; the script keeps slugs stable by deriving them from the same name rules. If you change display names in the source, only re-run the generator.
- **Manual edits:** Use the same rules as the pipeline: lowercase, spaces → `-`, only `a-z0-9-`, no accents (e.g. `sao-paulo`). Ensure the new slug is unique in `cities.json`. After changing a slug, consider redirects (e.g. in server or CDN) from the old slug to the new one if the page was already public.
- **Uniqueness:** The pipeline resolves duplicates by appending `-2`, `-3`, etc. If you add a city manually and its slug already exists, use a suffix (e.g. `london-canada`).

## Schema

Each city in `cities.json`:

| Field          | Type    | Required | Description                                      |
|----------------|---------|----------|--------------------------------------------------|
| city           | string  | yes      | Display name (e.g. "New York")                   |
| country        | string  | yes      | Country name                                     |
| slug           | string  | yes      | URL slug (e.g. "new-york")                       |
| timezone       | string  | yes      | IANA timezone (e.g. America/New_York)            |
| lat            | number  | no       | Latitude (for map + weather)                     |
| lng            | number  | no       | Longitude                                        |
| population     | number  | no       | Population (shown in "About [City]")              |
| state          | string  | no       | State/region (e.g. "New York")                   |
| continent      | string  | no       | Continent (breadcrumbs, About section)            |
| countryCode    | string  | no       | ISO 3166-1 alpha-2 (e.g. "US")                   |
| region         | string  | no       | Region (optional)                                |
| popular        | boolean | no       | If true, included in "Popular cities" links      |
| touristPlaces  | array   | no       | List of "Places to visit" (or use `places`)      |
| places         | array   | no       | Alias for touristPlaces                          |
| description    | string  | no       | Short description for "About [City]" (SEO)       |

Slugs must be unique and URL-safe (lowercase, hyphens). The server normalizes lookups to lowercase.

## Slug generation rules (pipeline)

- **City slug:** from ASCII name — lowercase, spaces → `-`, remove non–`a-z0-9-`, strip leading/trailing `-`. Accents removed (NFD + strip combining marks). Examples: New York → `new-york`, São Paulo → `sao-paulo`.
- **Country slug:** same rules applied to country name; used for `/country/:countrySlug` and breadcrumbs.
- **Uniqueness:** duplicate city slugs get numeric suffixes: `london`, `london-2`, etc.

## Data validation tips

- **Required:** `city`, `country`, `slug`, `timezone` must be non-empty.
- **Timezone:** must be a valid IANA timezone. Invalid zones break time display and time-difference pages.
- **Coordinates:** if present, `lat`/`lng` in [-90,90] and [-180,180]. Used for weather and sunrise/sunset.
- **Duplicate slugs:** no two cities may share the same `slug`; the first match wins in routes and sitemap.
- **Country consistency:** use the same country name spelling so country grouping works.

## Scaling and sources

- **Sitemap:** `/sitemap.xml` is generated from `cities.json`. Set `BASE_URL` in the environment for production.
- **Search:** `/api/search` and the global search bar read from this dataset; in-memory and fast at 5000+ cities.
- **Sources used by the pipeline:** GeoNames `cities5000.zip` (cities with population > 5000) and `countryInfo.txt` for country/continent. You can switch to `cities1000.zip` or `allCountries.zip` and adjust `MIN_POPULATION` to scale up or down.
